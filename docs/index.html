The Problem

Exceptions hide the truth

Traditional C# forces every caller to guess what a method might throw. MonadicSharp makes failures explicit in the return type — no more invisible control flow.

Exception-driven (before) C# 
// What does this throw? When?
// You have to read the whole call chain.
public User CreateUser(string name, string email)
{
    if (string.IsNullOrWhiteSpace(name))
        throw new ValidationException("Name required");

    if (!email.Contains('@'))
        throw new ValidationException("Invalid email");

    var existing = _db.Users
        .FirstOrDefault(u => u.Email == email);
    if (existing != null)
        throw new ConflictException("Email exists");

    return _db.Save(new User(name, email));
}

// Every caller needs a try/catch.
// Errors are invisible in the signature.
Railway-Oriented (after) C# 
// The signature tells the truth.
// Errors propagate automatically.
public Result<User> CreateUser(string name, string email)
{
    return ValidateName(name)
        .Bind(_ => ValidateEmail(email))
        .Bind(_ => CheckEmailNotTaken(email))
        .Map(_  => new User(name, email))
        .Bind(u => _db.Users.AddAsync(u));
}

// No exceptions. No hidden branches.
// Composable. Testable. Honest.

HTTP-mappable errors

// Semantic constructors → HTTP status codes
Error.Validation("Email invalid", "email")    // 400
Error.NotFound("Order", orderId.ToString())   // 404
Error.Conflict("Username already taken")      // 409
Error.Forbidden("Admin role required")        // 403

// Enrich with context at any stage
Error.Create("Payment gateway timeout")
    .WithMetadata("gatewayId", gateway.Id)
    .WithInnerError(originalException);

Collect ALL validation errors

// Stop at first error? No.
// Collect every failure in one pass.
var result = new[]
{
    ValidateName(request.Name),
    ValidateEmail(request.Email),
    ValidateAge(request.Age),
    ValidateRole(request.Role)
}.Sequence();

// result contains ALL errors, not just the first

MonadicSharp.AI

First-class LLM support

LLM APIs fail in predictable, typed ways. MonadicSharp.AI gives you typed errors and retry primitives that compose naturally with the rest of the pipeline.

⚡

Typed LLM Errors

AiError.RateLimit, ModelTimeout, InvalidOutput, TokenLimitExceeded — each with correct retry semantics baked in.

🔄

Automatic Retry

Exponential backoff for retriable errors, immediate failure for terminal ones — a single method call.

🔍

Execution Tracing

AgentResult.StartTrace gives a structured log of every step, its input, output, and duration. Plug into OpenTelemetry.

🛡️

Security Layer

MonadicSharp.Security detects prompt injection and masks secrets before they reach the LLM context window.

// Typed errors — never catch HttpRequestException
AiError.RateLimit()           // 429 → retriable, backoff
AiError.ModelTimeout()        // timeout → retriable
AiError.InvalidOutput()       // bad JSON → self-healable
AiError.TokenLimitExceeded()  // context full → terminal

// Retry with exponential backoff — one line
var result = await Result
    .TryAsync(() => _llm.CompleteAsync(prompt))
    .WithRetry(maxAttempts: 3,
               initialDelay: TimeSpan.FromSeconds(2));

// Multi-step agent with execution tracing
var agentResult = await AgentResult
    .StartTrace("SummarizeAgent", userInput)
    .Step("Retrieve",  q   => _search.FindChunksAsync(q))
    .Step("Generate",  ctx => _llm.CompleteAsync(ctx))
    .Step("Validate",  out => ParseAndValidate(out))
    .ExecuteAsync();

Quick Start

Up and running in 3 steps

No configuration, no boilerplate. Add the package and write your first pipeline in minutes.

1

Install

Add the core package to any .NET 8 project.

dotnet add package MonadicSharp
2

Write your first pipeline

Replace try/catch blocks with composable Result chains.

using MonadicSharp;

public Result<User> CreateUser(
    string name, string email) =>
    ValidateName(name)
        .Bind(_ => ValidateEmail(email))
        .Bind(_ => CheckNotTaken(email))
        .Map(_  => new User(name, email))
        .Bind(u => _repository.SaveAsync(u));
3

Match at the boundary

Unwrap Result only at the API layer.

app.MapPost("/users", async (
    CreateUserRequest req,
    UserService svc) =>
{
    var result = await svc.CreateUser(
        req.Name, req.Email);

    return result.Match(
        onSuccess: u => Results.Created(
            $"/users/{u.Id}", u),
        onFailure: e => e.Type switch
        {
            ErrorType.Validation =>
                Results.BadRequest(e),
            ErrorType.Conflict =>
                Results.Conflict(e),
            _ => Results.Problem(e.Message)
        });
});
